# Language Specification

The formal language specification lives at the repository root in `LANGUAGE-SPEC.md`.

This DocFX page is the public entry point for the spec. The current implementation keeps the full source document in place so compiler, runtime, tooling, and docs updates all point at the same contract.

Read the spec when you need:

- exact syntax rules
- parser and diagnostic expectations
- state semantics
- runtime reconciliation semantics
- tooling expectations
- v1 non-goals

For a shorter learning path, start with the [language overview](index.md), [concepts](concepts.md), and [syntax](syntax.md).

> A future cleanup can split the formal spec into smaller DocFX pages. Until then, `LANGUAGE-SPEC.md` remains the source of truth.
