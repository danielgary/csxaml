# Getting Started

Start here when adding CSXAML to an app or library for the first time.

Recommended order:

1. Read [Why CSXAML?](why-csxaml.md) if you are choosing between XAML, CSXAML, and handwritten C# UI.
2. Review [prerequisites](prerequisites.md).
3. If you do not already have a WinUI project, [create a new app](create-new-app.md).
4. Follow the [quick start](quick-start.md).
5. Build the [Todo tutorial](../tutorials/todo-app.md).
6. Keep [build troubleshooting](../troubleshooting/build-and-generation.md) nearby for package and generator issues.

Normal app authors install the `Csxaml` package. The package brings the build targets, packaged generator payload, and runtime dependency needed by generated components.
